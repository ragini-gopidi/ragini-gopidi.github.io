Healthcare ETL — Batch + Quality

Context: Built end-to-end pipelines to integrate clinical/operational data for analytics.

Design: Ingest → land in Azure Data Lake → transform in Databricks (PySpark) → model in Synapse/dbt → serve to BI.

  • Orchestrated with Azure Data Factory and Airflow; parameterized, idempotent runs.
  • Quality gates (schema, PK uniqueness, null thresholds) before loads; quarantine on failure.
  • Optimized with partitioning, file sizing, and incremental strategies for large tables.

Tech: Azure ADF, Databricks/PySpark, Synapse, dbt, Data Lake, Power BI.

Role: Data Engineer — pipeline design, modeling, DQ, and production support.
